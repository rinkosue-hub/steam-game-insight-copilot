# 2 分钟面试讲解稿

大家好，这个项目叫 Steam Game Insight Copilot，是我做的一个 Steam 玩家反馈与竞品分析工具。

它解决的问题是：游戏团队在做版本复盘、竞品调研或商店页优化时，需要阅读大量玩家评论，但手工整理非常慢，而且结论容易停留在“感觉”层面。所以我把这个流程做成了一个 Streamlit Web 应用。

用户可以输入 Steam 游戏名、AppID 或商店链接。系统会自动搜索游戏、分页获取 Steam 公开评论，然后做文本清洗、好评率统计、游玩时长统计和关键词提取。接着我设计了一套游戏业务标签规则，把评论归类到玩法体验、内容量、性能优化、价格、平衡性、BUG、美术音乐等维度。

项目里我最想强调的是“玩家情感 × 热度”四象限模型。横轴是某类反馈出现的热度，纵轴是该类反馈中的好评占比。这样可以把评论转化成五类业务结论：核心卖点、优先修复问题、潜力亮点、长尾问题和分歧反馈。它比单纯词频更接近真实工作场景，因为团队不仅要知道玩家在说什么，还要知道什么值得优先处理。

最后，系统会生成 Markdown 分析报告，包含核心结论、正负反馈、问题归因、四象限解读、P0/P1/P2 优化建议和后续观察指标。竞品页面还支持多款游戏对比，方便看不同产品的玩家认可点和吐槽点。

技术上使用 Python、Streamlit、requests、pandas、jieba、scikit-learn、plotly、BeautifulSoup 和 pytest。项目默认不依赖 OpenAI API Key，用模板也能完整运行；如果配置 Key，则可以进一步做自然语言润色。这个设计保证了它既能稳定演示，也能体现 AI 对信息收集、数据分析、方案整理和内容产出的效率提升。
